## Improvements Completed

- Converted internal Compassion, data-tools, donation, and dashboard-preview anchors to Next.js `Link` navigation.
- Restored unconditional React hook ordering in Steward message rows and the global module switcher.
- Moved Events summary and event-card components to stable module scope.

- Updated the in-app production-readiness source to reflect current validation evidence.

## Remaining Findings

- `Not Implemented`: module-level workspace permission enforcement is not consistent across Compassion, Events, HRM, and shared navigation boundaries.
- `Not Implemented`: payment-provider webhook processing and idempotent replay handling are absent.
- `Not Implemented`: unified authorization for uploads and exports is incomplete.
- `Not Implemented`: a tested backup/restore runbook is still missing.

### Medium priority

- `Partially Working`: 125 lint warnings remain. The most important are hook dependency warnings because they can produce stale closures; unused symbols and intentional email-rendering `<img>` warnings are lower risk.
- `Partially Working`: legacy and compatibility routes remain, including duplicate-looking route families and the `/oyama-email/callender` spelling compatibility surface. Canonical-route ownership should be verified before cleanup.
- `Partially Working`: several settings pages truthfully expose backend TODO states. They should remain gated until persistence APIs exist.

### Repository hygiene

Root-level scratch logs and older plan documents are present. No files were removed because deletion requires canonical replacement checks, reference scans, inbound-link updates, and a dedicated cleanup audit under repository governance.

## Recommended Next Slice
