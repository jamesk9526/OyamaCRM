# Phase 07 — Events and Gala Operations

## Goal

Extend the current events foundation into a complete registration, check-in, and revenue workflow.

## Already in place

- Event schema
- Event list/create/update APIs
- `/events` page scaffold

## Remaining scope

- Registration / ticketing
- Seating / host workflows
- Sponsorships
- Check-in
- Auction support
- Event reporting and volunteer-hour tie-ins

## Remaining implementation steps

1. Build registration and ticketing flows.
2. Add table seating, hosts, and sponsor tracking.
3. Build check-in and walk-in workflows.
4. Add auction support and post-event reporting.
5. Tie volunteer hours and event revenue into reporting.

## Exit criteria

- Staff can run a real event lifecycle from registration through reconciliation.
